# Self-Review — 자가 품질점검·콜 검증 루프

내가(데스크가) 내린 판단이 **실제로 맞았는지** 데이터로 후행검증한다. 검증된 방법론(33스킬 영상의 "매일 스스로
품질 점검·자가수정" 착안)을 정훈 포트폴리오에 맞춰, **콜 정확도 → 캘리브레이션 → 기준 교정**까지 닫는 루프.
투자 자문 아님 — 분석 참고, 결정은 정훈. (룰·전망을 자동으로 바꾸지 않고 **교정을 제안**한다. 확정은 정훈.)

## 언제 도나
- 정훈이 "콜 점검 / 내 분석 맞았어? / 별점 잘 맞췄나 / 스코어카드" 요청 시 즉시.
- **주말 캘리브레이션 루틴 R4(토 09:00) = 이 스킬의 1차 트리거**(`docs/routines.md`). 캘리브레이션 부채를 매주 청산.
- **주간 첫 보고서** 또는 큰 장세 변화 뒤 캘리브레이션이 필요할 때(portfolio-desk §7에서 연결). R4가 돌았으면 중복 생략 가능.
- 무거운 데스크 파이프라인은 안 돈다 — 과거 콜 vs 실제 비교에 집중.

## 0. 기계 채점 먼저 (자동화 — reflection 정량화)
1~3단계(콜 복원→실제 수집→채점)는 **자동화돼 있다.** 먼저 돌려 정량 베이스를 잡고, 사람이 §5 캘리브레이션 판단만 한다.
```bash
python3 .claude/skills/portfolio-desk/scripts/score_calls.py --backfill   # git 히스토리 → calls_log.jsonl (처음/재동기화 시)
python3 .claude/skills/portfolio-desk/scripts/score_calls.py              # 별점버킷 평균전진%·방향적중 + Brier 캘리브레이션 갭(과신 플래그)
python3 .claude/skills/portfolio-desk/scripts/hunter_score.py             # 경제사냥꾼 트랙레코드(검증/정정/미확인 추세 = 채널 신뢰도)
```
- 원장 = `data/app/calls_log.jsonl` (구조화된 콜의 시계열, git 히스토리에서 백필·보고서마다 `--append` 누적).
- **Brier proper score**(외부 예측연구 차용): 별점→내재확률(⭐5=.85…⭐1=.15) 매핑 후 실현방향 대비 Brier. 무정보(0.5고정=0.25)보다 **높으면 과신** 신호. 버킷별 '표현확신 vs 실제상승률' 갭으로 ⭐4~5 과신/⭐1~2 과소를 숫자로 본다.
- **hunter_score**: 채널 누적 [정정]률=수치 신뢰도, [검증]률=방향성 신뢰도의 대리지표 → §3 '경제사냥꾼 트랙레코드'를 수동→기계로 닫는다.
- 출력의 **별점 역전·과신·정정률 플래그**가 §3 캘리브레이션의 기계 답안지다. 아래 1~3은 그 결과를 사람이 해석·보강할 때 참조.

## 1. 과거 콜 복원 (머신 정본 우선)
`data/app/stocks.json`은 보고서마다 종목별 `stars`·`target`·`buy_zone`·`trim`·`as_of`·`source_report`로 갱신된다.
git 히스토리가 과거 콜의 타임머신이다 (score_calls.py가 이걸 자동 백필).
```bash
# 특정 보고서 시점의 콜 스냅샷 복원 (예: 2~3주 전 커밋)
git log --oneline -- data/app/stocks.json | head -15
git show <commit>:data/app/stocks.json    # 그 시점의 별점·목표·매수존
```
- 머신 정본이 없던 옛 시점은 `docs/reports/report_v*.md`의 보유 풀표에서 별점·목표·매수존을 읽어 복원.
- 검증 대상: **별점/스코어**(방향성), **매수존**(존 진입 후 반등했나), **목표가**(방향·진척), 경제사냥꾼 주장(이미 [검증/정정/미확인]).

## 2. 실제 결과 수집
```bash
python3 .claude/skills/portfolio-desk/scripts/market_data.py --tickers <티커들>   # 현재가
```
- 콜 시점 가격(과거 스냅샷) → 현재가로 **실현 경로**를 잡는다. 환율은 fx 그룹값.
- 매수존: 콜 이후 실제로 그 가격대에 닿았는지, 닿고 반등/이탈했는지.

## 3. 채점 (캘리브레이션)
- **별점 캘리브레이션**: 별점 4~5 콜들의 평균 이후수익률 vs 별점 1~2 콜들 — 높은 별점이 실제로 더 좋았나(순서가 맞나)가 핵심. 역전되면 채점기준 편향 신호.
- **매수존 적중률**: 제시한 존에 닿은 비율 / 닿고 반등한 비율.
- **목표가 방향성**: 목표 방향(상/하향)이 실제 방향과 일치한 비율. (정확한 가격 도달이 아니라 **방향·진척** 기준.)
- **경제사냥꾼 트랙레코드**: `docs/research/hunter_log.md`의 [정정] 비율 — 채널이 방향성은 좋아도 수치 과장 잦음(기지정 사실) 재확인.

## 4. 스코어카드 기록 (시계열 누적)
score_calls.py 정량 결과 + 사람 해석을 `docs/research/call_scorecard.md` **맨 위에 prepend**(누적 = 캘리브레이션 추세).
(원장 calls_log.jsonl이 기계 시계열, scorecard.md는 사람의 해석·교정 메모):
```
## 콜 검증 {YYYY-MM-DD} (대상 기간 {시작}~{끝})
- 별점 캘리브레이션: ⭐4~5 평균 {x}% vs ⭐1~2 평균 {y}% → {순서 정상/역전}
- 매수존 적중: 닿음 {a}/{n}, 닿고 반등 {b}/{a}
- 목표가 방향성: {맞음}/{전체}
- 경제사냥꾼: [검증]{p} [정정]{q} [미확인]{r}
- 빗나간 콜 Top: {종목·콜·실제·원인 한 줄}
```

## 5. 자가수정 — 교정 제안 (자동 변경 ❌)
체계적 편향이 보이면 **구체적 교정안**을 PM/정훈에게 제시(영상의 "90점 미만이면 코드 수정" 착안, 단 정훈 승인 후 반영):
- 별점이 한쪽으로 쏠리거나(전부 4~5) 캘리브레이션 역전 → `stock-deepdive`의 방법론 체크리스트·환산기준 조정 제안.
- 특정 종목 전망이 반복해서 빗나감 → `docs/holdings_outlook.md` 해당 종목 전망 갱신 제안.
- 사실 오류 발견 → 즉시 `docs/master.md` §7 영구교정 + STATE SNAPSHOT(추측 아닌 검증된 정정만).
- 변경은 정훈 확인 후. 검증된 정정(사실 오류)만 예외적으로 바로 기록.

## 규약
- 후행검증은 **결과론 함정** 주의: 단기 노이즈로 콜을 과벌하지 말 것 — 표본·기간 명시, "운 vs 실력" 구분.
- 추측 금지, 미확인은 "미확인". 스코어카드 갱신·교정 시 git 커밋(연속성).
